== Overview

The Programmable Packet Processor Complex (PPC) is

* An high-performance
* Fully programmable networking engine

The PPC architecture is optimized for a Partial Configurable Stage model, allowing for extreme flexibility in protocol handling while maintaining strict hardware-level performance guarantees.

Key Capabilities:

* **Packet Parsing:** Parsing up-to `64` Bytes, supporting L2-L3 Protocols
* **Configurable Ordering Domain**
* **Classification:** Multi-Engine Packet Classification: EM, WCM, LPM Algorithem
* **Traffic Engineering:** Support for Mirroring, Multicasting, and Hashing.
* **Packet Modification:** Supporting Encapsulation, Decapsulation, NAT via INSERT, REMOVE and REPLACE actions.

This packet processsor is P4 Configurable.

=== PPM
Packet Processor Manager is the orchestration hub of the complex, managing the lifecycle of a packet within the chip.


==== IQM
Input Queue Manager - Ingress buffer.
Hold Packet Information in an In-Order Queue until a PMU dequeue it.

==== PMU
Packet Management unit.
The PPC utilizes a multi-threaded processing model.
Each PMU acts as an independent thread of execution.

* **Multi-Stage Processing:** Each PMU thread can be configured to execute up to 4 sequential stages of processing per packet.
* **PFE Engine Mapping:** In each stage, the PMU can trigger one or more engines within the PFE.
* **Parallel Engine Operation:** If multiple engines are configured for a single stage, the PMU initiates these lookups/operations in parallel (VLIW-style), significantly reducing the total clock cycles required for complex header manipulations.
* **State Management:** While lookups are offloaded to the PFE, the PMU maintains the packet's local architectural state and metadata between stages.

PMU deside the Stages setup for a given packet via the `PMD.COMMON.PKT_FLOW`.

The PMU goes stage by stage:

* if `PPC_PPM_PMU_STATE_CONFIG_<#>[PMD.COMMON.PKT_FLOW].STAGE_<##>_EN` == 1
** Send to RMC `PPC_PPM_PMU_STATE_CONFIG_<#>[PMD.COMMON.PKT_FLOW].STAGE_<##>`
** Wait for complition
* else
** Notify OQM: `PKT_READY`

==== RMC
Resource Manager Control

Manage PFE engine resources by allocating between PMU threada.

* Manages contention for shared engines across all active PMU threads.
* Handles the request/response bus protocol, ensuring results are returned to the correct PMU thread and stage.

==== ODQ
Ordering Domain Queue

* Stores packets according to there respective ordering domain
* Head of line blocking - to save the original ordering within a given domain.

==== OQM
Ordering Domain Manager - the control logic that triggers the release of packets to the output stage once order is restored.

=== PFE
Processor Fabric Engine.

Composed of

* 4 Exact Match (EM) Classifer Engines
* 2 Wild-Card Match (WCM) Classifer Engines
* 1 Longest Prefix Match (LPM) Classifer Engine
* 1 Range Checker Engine
* 1 Modifier Engine
* 1 Hashing Engine

==== IFE
Input Fabric Engine - the ingress interface to the PFE

* handle request buffering 
* distribute request to specific engines

==== EM CLS Engine
The Exact Match (EM) Classifer Engines processes up to 64-bytes packet header. It computes a hash, performs a multi-way lookup in a hash table (SRAM-based), and returns a Action.

* Configurable Extractor - extract Key from packet
* Look-Up Engine
** Hash Based for Key to Address mapping
*** Hash Algo: Cuckoo Hash or SHA256 Hash
** Look-Up Table

===== Extractor

Performs bit-masking on the 64-byte input to isolate the predefined 5-tuple (or custom flow-key). It aligns fields into a flat "Search Key."

===== Hash Engine
Uses a hardware-accelerated CRC-32 or Toeplitz implementation.
To minimize collisions, **Dual-Hash** approach (generating two indices) to map into a Cuckoo Hashing table or a chained bucket structure.

===== Lookup Engine

** Phase 1: Indexing SRAMs using the hash result.
** Phase 2: Comparing the stored key in the bucket against the incoming Search Key.
** Phase 3: Hit/Miss logic and Action Pointer retrieval.

==== WCM Engine
Wild-Card Match (WCM) Classifer Engines:

* Configurable Extractor - extract Key from packet
* Look-Up Engine
** TCAM based Matching for Key to Address mapping
** Look-Up Table

Use-Case: ACLs and complex policy enforcement

==== LPM Engine
Longest Prefix Match (LPM) Classifer Engine

* Configurable Extractor - extract Key from packet
* Look-Up Engine
** LPM Algorithem Maching for Key to Address mapping
** Look-Up Table

Use-Case: L3 routing table lookups
